Android Studio配置使用

2018/3/25
  • 一、配置
  • 二、使用
  • 三、补充

一、配置

  • 配置文件目录

    • Configuration Path：~/Library/Preferences/<PRODUCT><VERSION>，删除该目录即还原默认配置
    • Caches Path：~/Library/Caches/<PRODUCT><VERSION>
    • Logs Path：~/Library/Logs/<PRODUCT><VERSION>

  • 禁止AS首次启动下载SDK：显示包内容，bin目录下的idea.properties增加disable.android.first.run=true；

  • Apperance&Behavior

    • 主题（程序外观、文本编辑器）： Appearance --> 修改Themes为Darcula
    • 显示内存使用：Appearance --> 勾选Show memory indicator
    • 关闭打开上次工程：System Settings --> 取消Reopen last...
    • 关闭自动更新：System Settings --> Updates --> 取消Automatically check updates...

  • Editor

    • API文档悬浮提示：General --> 勾选Show quick documentation...
    • 自动导包：General --> Auto Import --> 勾选Add unambigous...、Optimize...
    • 显示代码行数：General --> Appearance --> 勾选Show line numbers
    • 代码自动提示（新版已默认）：General --> Code Completion --> Autopopup code completion，取消大小写敏感，设置Case sensitive completion为None
    • 禁止代码自动折叠：General --> Code Folding --> 取消One-line methods、Closues...、Generic constructor...
    • 文本编辑器字体：Colors&Fonts --> 修改Font（新版的内置Scheme也可直接修改不需要复制了）
    • Console字体：Colors&Fonts --> 修改Console Font
    • 主题（文本编辑器）：Colors&Fonts --> 修改Scheme，第三方主题https://github.com/rainglow/jetbrains
    • 文件编码：File Endcodings（默认UTF-8）

二、使用

  • 工程区别（AS --> Eclipse）：project --> workspace、module --> project；

  • AS 2.1版本开始，新建的Activity默认继承AppCompatActivity；

  • 删除module：需要先Open Module Settings删除对应module，之后右键才有delete选项，之后进行删除；

  • 代码自动换行：View --> Active Editor --> Use soft Wraps；

三、补充

  • 各种Version

    • a、compileSdkVersion
      • 使用哪个版本的SDK API进行构建项目；
      • 其对应Android SDK目录下的platforms目录下的版本；
    • b、buildeToolVersion
      • 构建项目时，所使用的构建工具的版本；
      • 其对应其对应Android SDK目录下的build-tools目录下的版本；
      • 一般每个Android版本都会有对应版本的buildTools；
      • 新版本的SDK API添加了新特性，需要新版本的构建工具来进行构建，老版本的构建工具不能构建；
      • 新版本的构建工具可以兼容构建老版本的SDK API；
      • 有时Android Gradle Plugin版本更新，需要把buildToolsVersion提升至指定的版本才可以继续使用；
      • Android Studio 3.0开始，隐藏了buildToolsVersion这个配置，默认使用最新版本的构建工具；

    • c、minSdkVersion

      • App最低支持的手机版本，低于这个版本的设备无法安装；
      • 若minSdkVersion设置成12，但代码使用了一个SDK API 20才有的方法，此时编译器会发出警告，允许在低于20的设备上会崩溃；

      • 可以使用以下方式解决

        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.LOLLIPOP) {
    // 在Android 5.1（API>=21）的设备上
} else {

}

    • d、targetSdkVersion

      • 设置SDK API运行时的目标版本；其是Android为了适应系统版本更新过快，提供的一种向下兼容的方案；
      • 如权限检查机制在Android 6.0改成了动态申请，需要手动在项目中调用申请权限的代码；若将targetSdkVersion改为低于Android 6.0，则权限检查机制则不会使用新特性来动态申请；

    • e、Gradle Version

      • Gradle是一个构建项目的工具，可以把文件、文件夹按照一定的规则关联起来形成一个项目，其不仅仅用在Android Studio中；

    • f、Android Plugin Version

      • 在Android Studio中使用的Gradle其实是Android Gradle Plugin，简称Android Plugin（安卓插件）；
      • Android Studio的构建系统基于Gradle，并且Android Gradle Plugin添加了构建Android应用程序特有的功能；
      • Android Plugin通常与Android Studio一起更新，但其和Gradle是可以独立于Android Studio运行，并可以单独更新；
      • Android Plugin也有版本号，而每个版本会限定使用的Gradle的版本；若这两个版本对应不上，则工程构建报错；
      • 所以，Android Plugin Version、Gradle Version、buildeToolVersion版本是相互关联的；（https://developer.android.com/studio/releases/gradle-plugin）
      • https://www.cnblogs.com/whycxb/p/9111254.html

  • SDK目录文件

    • a、platforms
      • 存放不同API Level版本SDK的地方；（目录内按版本号区分）
      • AS中下载SDK Platform；
    • b、build-tools
      • 不同版本的构建工具，如aapt、dx、aidl等；（目录内按版本号区分）
      • AS中下载Android SDK Build-Tools；
    • c、platform-tools
      • 一些Android平台相关的工具，如adb、fastboot等；（目录内不按版本号区分）
      • AS中下载Android SDK Platform-Tools；
    • d、tools
      • 一些Android开发相关的工具，如ddms、draw9patch等；（目录内不按版本号区分）
      • AS中下载Android SDK Tools；

  • 引入第三方库

    • a. 通过将第三方jar包复制到工程的libs目录下，然后选中jar包，之后进行grandle同步；
    • b. 通过配置Gradle导入（易于升级）：右键Open Module Settings --> Dependencies，点击 + 后，选择Library dependency，其会自动下载相关的lib；
    • 修改gradle配置后，需要进行手动同步；
