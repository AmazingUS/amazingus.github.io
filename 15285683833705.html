Java、ObjC、Swift中的错误、异常的处理总结

2018/5/10
  • 一、Java
  • 二、ObjC
  • 三、Swift

一、Java

  • 在Java中，所有的错误、异常的最上层父类为Throwable，其有两个重要的子类：Error（错误）、Exception（异常）；

    • a. Error（错误）：程序无法处理，大多数为JVM出现的问题；比如JVM运行错误（VirtualMachineError）、当 JVM没有内存资源错误（OutOfMemoryError）等；
    • b. Exception（异常）：程序可以处理，这是与Error（错误）的本质区别；

  • Exception（异常）也分为两种：运行时异常、非运行时异常（编译异常）；

    • a. 运行时异常：

      • a). 在编译时不会被检查；一般是由程序逻辑问题引起的，应该从逻辑角度尽可能避免这类异常的发生；
      • b) 程序中可以选择处理（try-catch-finally、throws），或者不处理异常；
      • c) 其都是RuntimeException及其子类，比如NullPointerException（空指针异常）、IndexOutOfBoundsException（下标越界异常）等；

    • b. 非运行时异常（编译异常）：

      • a) 必须要进行处理（try-catch-finally、throws）的异常，若不处理，程序就不能编译通过；比如IOException、SQLException等
      • b）除RuntimeException及其子类以外的异常都是非运行时异常（编译异常）；

  • 通常也将可以把Java的错误、异常分为：可查的异常（checked exceptions，也就是上面的非运行时异常）、不可查的异常（unchecked exceptions，包含错误和运行时异常）；

  • 处理异常有两种方式：捕获异常（try-catch-finally）、捕获异常后再抛出（throws）；

二、ObjC

  • 在ObjC中，所有的错误、异常的分为两种：NSError（错误）、NSException（异常）；

    • a. NSError（错误）：

      • 可以预见到发生的、可以恢复的非致命错误，比如找不到指定位置的文件、网络无法连接等；
      • 其相当于JAVA中的可查的异常（checked exceptions)；
      • 只能使用NSError *类型的指针去捕获处理错误的内容；

    • b. NSException（异常）：

      • 无法预见到发生的，通常也不可恢复的致命错误，程序会奔溃，比如磁盘坏了、内存没了、数组越界、无效的方法等；
      • 其相当于JAVA中的不可查的异常（unchecked exceptions）；
      • 只能使用@try-@catch-@finally去捕获处理，但是Apple不推荐使用，通常也不去处理）
      • 而对于JAVA来说，不论是哪种问题都使用try-catch-finally去捕获处理；

三、Swift

  • 在Swift中，NSError（错误）被Error（错误）替代；但是Error（错误）的处理方式有两种：

    • 捕获异常：do-try-catch、try?、try!；
    • 捕获异常后再抛出：throw；方法定义使用throws、rethrows抛出异常；

  • Swift中每一行的Error都需要使用try标记；

    do {
    let session = AVAudioSession.sharedInstance()
    try session.setCategory(AVAudioSessionCategoryMultiRoute)
    try session.overrideOutputAudioPort(.none)
    try session.setActive(true)
} catch let error {
    print(error)
}
