Logrotate配置使用

  • 一、概述
  • 二、Crontab（定时任务）
  • 三、配置案例
  • 三、其他日志切割方法

一、概述

  • Logrotate是一个非常有用的管理日志工具，其可以自动对日志进行切割（轮循）、压缩以及删除旧的日志文件；比如让设置logrotate自动化运作，让/var/log/acc日志文件每30天轮循，并删除超过6个月的日志；

  • Linux默认安装了Logrotate，其默认的配置文件为：/etc/logrotate.conf、/etc/logrotate.d/目录；

    • logrotate.conf为主要的配置文件，logrotate.d是一个目录，该目录中的所有文件默认都会被导入到/etc/logrotate.conf中（include /etc/logrotate.d），在logrotate.conf被logrotate时，同时也被logrotate；
    • 若/etc/logrotate.d/里面的文件中没有配置一些设置，则使用/etc/logrotate.conf中设置的默认值；
    • 对于多数Service，即使系统没有安装Logrotate，一个默认的logrotate配置文件会在安装程序包时自动创建；若是使用源代码编译手动安装，就需要手动在/etc/logrotate.d/目录中手动创建一个logrotate配置文件；

Nginx+uWSGI运行Python Web应用

  • 一、uWSGI基本使用
  • 二、uWSGI运行Flask应用
  • 三、Nginx+uWSGI配合使用

一、uWSGI基本使用

  • 新建一个python文件，定义一个application函数（其实就是wsgi的处理函数）；

    # server.py
def application(env, response):
    response('200 OK', [('Content-Type', 'text/html')])
    return [b'<h1>Hello,Web!</h1>'] # 注意: 返回的是字节

  • 终端中运行：uwsgi --http :9090 --wsgi-file server.py ，之后就可以访问http://127.0.0.1:9090/；

    • a. --http参数指定HTTP监听地址端口；
    • b. --wsgi-file参数指定WSGI应用程序入口；
    • c. uWSGI会自动搜寻名为”application”的应用对象并调用，也可以使用--callable参数修改该默认名称；

Python Fabric模块

  • 一、概述
  • 二、使用
  • 三、案例

一、概述

  • Fabric是一个Python库，其提供了丰富的与SSH交互的接口，可以用来在本地或远程机器上自动化、流水化地执行Shell命令，因此非常适合用来做应用的远程部署及系统维护；

  • Fabric分为Python2和Python3版本的，安装Python3版本：pip install fabric3；

  • fab是Fabric程序的命令行执行命令，默认执行一个fabfile.py的文件（若需执行其他文件，使用-f参数指定文件即可）；

    • fabfile.py文件中每个函数就是一个任务，任务名即函数名；

    • fab命令就是用来执行fabfile.py中定义的任务，且必须显式地指定任务名；可以使用-l参数列举当前fabfile.py文件中定义了哪些任务；

    • fab命令语法为 fab [options] <command> [arg1,arg2:val2,host=foo] ；
