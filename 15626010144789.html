iOS状态栏

2019/7/8
  • 一、启动时设置
  • 二、启动后设置
    • 1、方式1：全局设置（iOS 9开始废弃）
    • 2、方式2：控制器单独设置（iOS 7开始支持）
    • 3、补充：当前控制器是UINavigationController的子控制器

一、启动时设置

  • 可以设置Target的Status Bar，可以设置Info.Plist，这两个地方是一样的；

    // 设置启动时是否显示状态栏
UIStatusBarHidden：Status bar is initially hidden
// 设置启动时状态栏样式
UIStatusBarStyle：Status bar style

二、启动后设置

1、方式1：全局设置（iOS 9开始废弃）

  • 启用：

    • 配置info.plist的View controller-based status bar appearance项为NO，不添加该项默认值为YES；

  • 设置：

    // 设置状态栏显示
UIApplication.shared.isStatusBarHidden = true

// 设置状态栏样式
UIApplication.shared.statusBarStyle = .default
    • 注意：若启用后没有进行设置，则继续使用启动时设置的配置；
2、方式2：控制器单独设置（iOS 7开始支持）

  • 启用：

    • 配置info.plist的View controller-based status bar appearance项为YES，或者不添加该项（默认值为YES）；

  • 设置：

    // 设置状态栏显示隐藏
// 默认值：竖屏VC的为false、横屏VC的为true
open var prefersStatusBarHidden: Bool { get }

// 设置状态栏样式
// 默认值为default
open var preferredStatusBarStyle: UIStatusBarStyle { get }

    // 设置状态栏显示、消失时的动画类型
open var preferredStatusBarUpdateAnimation: UIStatusBarAnimation { get }

// 状态栏样式改变后，需要调用该方法，重绘状态栏
open func setNeedsStatusBarAppearanceUpdate()

    • a. 横屏模式下，系统会自动将状态栏会隐藏，可以重写prefersStatusBarHidden返回true，让其一直显示；

      • 注意：iOS 11，iPhone X被设置横屏模式强制隐藏，重写无效；iOS 13，所有iPhone被设置横屏模式强制隐藏，重写无效；
      • https://stackoverflow.com/questions/46666603/status-bar-in-landscape-for-iphone-x；
      • https://stackoverflow.com/questions/57590252/ios-13-status-bar-hidden-for-all-iphone-types-in-landscape；

    • b. 若当前控制器是UINavigationController的子控制器，则当前控制器的preferredStatusBarXX方法不会被调用，导航控制器的preferredStatusBarXX才具有修改状态栏的能力；有两种方式可以解决（以Style为例）：

      • a) 方式1：重写UINavigationController的preferredStatusBarStyle，直接返回当前显示VC的preferredStatusBarStyle；

        // BaseNavigationController
override var preferredStatusBarStyle: UIStatusBarStyle {
    return self.topViewController?.preferredStatusBarStyle ?? .default
}

      • b) 方式2：重写UINavigationController的childViewControllerForStatusBarStyle，直接返回当前显示VC；

        override var childForStatusBarStyle: UIViewController? {
    return self.topViewController
}
        • 此时，不再调用UINavigationController的preferredStatusBarStyle，而是去调用topViewController的preferredStatusBarStyle；
3、补充：当前控制器是UINavigationController的子控制器

  • 1）当Navigation Bar不可见时，由栈顶的Content ViewController负责管理Status Bar Style；

  • 2）当Navigation Bar可见时，由NavigationController负责管理Status Bar Style；NavigationController往往都是系统默认的UINavigationController，若Status Bar Style不符合要求，有两种解决方式：

    • a. 继承UINavigationController，重写preferredStatusBarStyle属性；
    • b. 不继承UINavigationController（没有办法继承，如系统组件MFMailComposeViewController）:
      • a）Navigation Bar的barStyle属性决定外观，修改barStyle会联动改变Status Bar Style：
      • b）当barStyle = .default时，Navigation Bar显示为黑色，此时Status Bar显示为白色；
      • c）当barStyle = .black时，Navigation Bar显示为白色，此时Status Bar显示为黑色；
      • d）设置barStyle后，会改变Navigation Bar的颜色，因此需要先修改barStyle，再设置barTintColor；（通过barTintColor可以自定义Navigation Bar的外观）

  • 3）通过Modal Presentation呈现的VC，若也是嵌套在NavigationController中，规则和上面的描述一致；

    • 但是只有当modalPresentationStyle为.fullScreen（全屏Presentation），呈现出的视图才对Status Bar Style有发言权，否则还是由原先的ViewController控制；
    • 若非要在不全屏Presentation的ViewController中控制Status Bar Style，通过设置modalPresentationCapturesStatusBarAppearance为true解决；

  • 4）参考：https://www.jianshu.com/p/4008c08a2df2；
