一、安装配置（C++、Python）

  • 1、安装OpenCV

    brew install opencv
    • 安装路径：/usr/local/Cellar/opencv（M1：/opt/homebrew/Cellar/opencv）；

  • 2、Xcode新建Command Line Tool工程，并支持C++；

  • 3、配置Header Search Paths、Library Search Paths等信息；

    • 1）方式一：（引入所有库）

      • a. 设置Header Search Paths；

        $(PROJECT_DIR)  // 当前工程路径
/opt/homebrew/Cellar/opencv/4.7.0_1/include/opencv4  // OpenCV的头文件路径
        • 注意：OpenCV的头文件路径不要以.../opencv4/opencv2结尾，否则#include "opencv2/opencv_modules.hpp"报错file not found；

      • b. 将OpenCV的库文件夹，拖拽到当前项目中；

        • 不勾选Copy items if needed，勾选Create groups和Add to targets；
        • OpenCV的库文件夹路径：/opt/homebrew/Cellar/opencv/4.7.0_1/lib；

      • 参考：

        • OpenCV在macOS下的开发环境搭建：https://blog.csdn.net/lxl122412/article/details/114472759

    • 2）方式二：（按需引入库）

      • a. 设置Header Search Paths；

        • 和方式一中的设置一样；

      • b. 设置Library Search Paths；

        $(PROJECT_DIR)  // 当前工程路径
/opt/homebrew/Cellar/opencv/4.7.0_1/lib  // OpenCV的库文件路径

      • c. 设置Link Binary With Libraries；

        • 点击+选择Add Other... -》Add Files...，CMD + Shift + G直接定位到/opt/homebrew/Cellar/opencv/4.7.0_1/lib ；
        • CMD多选，不带版本号的，并且后缀为.dylib的库文件；（只选几个常用的库文件即可）；
          • core、imagecodecs、highgui、impgproc

    • 3）方式三：（按需引入库；通过.xcconfig配置文件 + pkg-config✅）

      • a、运行pkg-config，获取链接器符号（Linker Flags）等信息；

        # 安装pkg-config（安装OpenCV时会自动安装）
brew install pkg-config

# OpenCV自带的opencv.pc文件路径为XX/Cellar/opencv/<版本号>/lib/pkgconfig/opencv.pc

# 获取链接器符号（Linker Flags）等信息
pkg-config --cflags --libs /opt/homebrew/Cellar/opencv/4.7.0_1/lib/pkgconfig/opencv4.pc 

        # 输出结果包含`Header Search Paths`（HEADER_SEARCH_PATHS）、
`Library Search Paths`（LIBRARY_SEARCH_PATHS）、
`Other Linkers Flags`（OTHER_LDFLAGS）
-I/opt/homebrew/opt/opencv/include/opencv4 -L/opt/homebrew/opt/opencv/lib -lopencv_gapi -lopencv_stitching -lopencv_alphamat -lopencv_aruco -lopencv_barcode -lopencv_bgsegm -lopencv_bioinspired -lopencv_ccalib -lopencv_dnn_objdetect -lopencv_dnn_superres -lopencv_dpm -lopencv_face -lopencv_freetype -lopencv_fuzzy -lopencv_hfs -lopencv_img_hash -lopencv_intensity_transform -lopencv_line_descriptor -lopencv_mcc -lopencv_quality -lopencv_rapid -lopencv_reg -lopencv_rgbd -lopencv_saliency -lopencv_sfm -lopencv_stereo -lopencv_structured_light -lopencv_phase_unwrapping -lopencv_superres -lopencv_optflow -lopencv_surface_matching -lopencv_tracking -lopencv_highgui -lopencv_datasets -lopencv_text -lopencv_plot -lopencv_videostab -lopencv_videoio -lopencv_viz -lopencv_wechat_qrcode -lopencv_xfeatures2d -lopencv_shape -lopencv_ml -lopencv_ximgproc -lopencv_video -lopencv_xobjdetect -lopencv_objdetect -lopencv_calib3d -lopencv_imgcodecs -lopencv_features2d -lopencv_dnn -lopencv_flann -lopencv_xphoto -lopencv_photo -lopencv_imgproc -lopencv_core
        • pkg-config是在编译应用程序和库时使用的辅助工具，可以帮助找到库（如OpenCV）的链接器符号（Linker Flags）等信息；

      • b、新建.xcconfig配置文件，配置Header Search Paths（HEADER_SEARCH_PATHS）、Library Search Paths（LIBRARY_SEARCH_PATHS）、Other Linkers Flags（OTHER_LDFLAGS）；

        HEADER_SEARCH_PATHS = $(inherited) /opt/homebrew/opt/opencv/include/opencv4

LIBRARY_SEARCH_PATHS = $(inherited) /opt/homebrew/opt/opencv/lib

OTHER_LDFLAGS = $(inherited) -lopencv_core -lopencv_imgcodecs -lopencv_highgui  -lopencv_imgproc

  • 4、测试代码（C++）

    #include <iostream>
#include <opencv2/opencv.hpp>

using namespace cv;

int main(int argc, const char * argv[]) {
    std::string imagePath = "/Users/scott/Desktop/a.png";
    
    Mat image = imread(imagePath, IMREAD_COLOR);
    
    if (!image.data) {
        std::cout <<  "Image does not exist!" << std::endl;
        return -1;
    }
    
    namedWindow("Display Image", WINDOW_AUTOSIZE);
    imshow("Display Image", image);
    
    Mat gray;
    cvtColor(image, gray, COLOR_RGBA2GRAY);
    
    namedWindow("Display Gray Image");
    imshow("Display Gray Image", gray);
    
    waitKey(0); // // 按键等待，需要在窗口任意输入字符退出；0代表永久等待
    
    return 0;
}
    • 补充：
      • 文件路径可以使用“相对路径”，此时需要修改Scheme中的配置（Edit Scheme -> Run中 -> Options -> Use custom working directory选项，然后自定义项目目录）；
      • 屏蔽OpenCV代码注释警告：Build Settings中的Documentation Comments配置为NO；
      • OpenCV使用imshow()展示图像时，即使没有使用nameWindow()，也能正常显示；若希望之后可以使用相同的窗口执行某些操作（例如移动、调整大小、关闭该窗口），需要通过nameWindow()使用相同名称，引用它来执行此操作；
        • OpenCV：在imshow() 之前使用namedWindow() 的必要性讨论？：https://dontla.blog.csdn.net/article/details/122809469

  • 5、支持Python3

    • 若使用brew安装了OpenCV，再使用pip直接安装opencv-python或opencv-contrib-python，会造成重复安装，brew安装的OpenCV本身就可以支持Python；

      # opencv-python只包含了主要模块的包；
pip install opencv-python

# opencv-contrib-python包含了主要模块以及扩展模块；
# 扩展模块主要是一些带专利的收费算法（如shift特征检测），
# 以及一些在测试的新的算法（稳定后会合并到主要模块）；
pip install opencv-contrib-python

    • 方式：将OpenCV安装路径下（XX/Cellar/opencv/<版本号>/lib/pythonXX/site-packages/XX）的cv2.cpython-XX-darwin.so文件，软链接到所需要的Python环境下的site-packages目录，并重命名为cv.so；

      # .so文件的具体路径
/opt/homebrew/Cellar/opencv/4.7.0_1/lib/python3.11/site-packages/cv2/python-3.11/cv2.cpython-311-darwin.so

# 所需要的Python环境下的site-packages目录的具体路径
/Users/scott/.pyenv/versions/3.11.1/lib/python3.11/site-packages

# 进行软链接
ln -s /opt/homebrew/Cellar/opencv/4.7.0_1/lib/python3.11/site-packages/cv2/python-3.11/cv2.cpython-311-darwin.so /Users/scott/.pyenv/versions/3.11.1/lib/python3.11/site-packages

二、补充

1、OpenCV的常见模块
  • 1）core
    • 核心功能模块，主要包含OpenCV库的基础数据结构、基本操作；
  • 2）imgcodecs
    • 图像文件读取与保存模块，主要用于图像文件读取与保存；
  • 3）highgui
    • 高层GUI，包含创建和操作显示图像的窗口、处理鼠标事件及键盘命令、提高图形交互可视化界面等；
  • 4）imgproc
    • 该模块名称是由image和process两个组合而成，是重要的图像处理模块，主要包含：图像滤波、几何变换、直方图、特征检测、目标检测等；
  • 5）ml
    • 机器学习模块，主要包括：统计分类、回归、数据聚类等；
  • 参考：
    • 【OpenCV之路】了解 OpenCV 的模块：https://blog.csdn.net/magic_ll/article/details/122252025
2、OpenCV的Mat

  • 1）属性、方法：

    • mat.dims：Mat矩阵的维数，若为二维矩阵，则值为2；

    • mat.rows：Mat矩阵的行数；

    • mat.cols：Mat矩阵的列数；

    • mat.size：Mat矩阵的大小；mat.size.width==mat.cols，mat.size.height==mat.rows；

    • mat.channels()：Mat矩阵中的单个元素的通道数；如RGB像素点，值为3；

    • mat.depth()：Mat矩阵中的单个元素的通道的精度； 值为0~7的数字，分别代表不同的位数，对应关系如下：

      #define CV_8U   0
#define CV_8S   1
#define CV_16U  2
#define CV_16S  3
#define CV_32S  4
#define CV_32F  5
#define CV_64F  6
#define CV_16F  7

    • mat.elemSize()：Mat矩阵中的单个元素的，数据大小的个数；

      • 以8位（一个字节）为一个单位，乘以通道数和8位的整数倍；
      • Mat中的数据类型是CV_8UC1，则值为1；CV_8UC3或CV_8SC3，则值为3；CV_16UC3或CV_16SC3，则值为6；

    • mat.elemSize1()：Mat矩阵中的单个元素的，其中一个通道的，数据大小的个数；

      • mat.elemSize1() = mat.elemSize() / mat.channels()

    • mat.step：Mat矩阵的每一行的，数据大小的个数，可以理解为“步长”；

    • mat.type()：；

  • 2）其他：

    • 对于RGB，0000000表示黑，FFFFFF表示白；对于GRAY，0表示白，255表示黑；
3、iOS中使用OpenCV注意事项（Apple M1设备，模拟器运行）
platform :ios, '15.0'
use_frameworks!
source 'https://github.com/CocoaPods/Specs.git'

inhibit_all_warnings!

target 'OpenCViOSDemo' do
  
  # Pods for OpenCViOSDemo
  pod 'PureLayout'
  pod 'OpenCV'
  
end

post_install do |installer|
  installer.pods_project.targets.each do |target|
    
    target.build_configurations.each do |config|
      config.build_settings['IPHONEOS_DEPLOYMENT_TARGET'] = '11.0'
      
      # 问题：
      # pod OpenCV引入是静态库.framework（armv7 armv7s i386 x86_64 arm64），
      # 模拟器编译后，App的二进制文件变成x86_64架构（即使Xcode不是Rosetta 2模式），
      # 这将导致PureLayout（被编译成arm6架构）无法被正常使用，
      # 模拟器运行报错：-[UICollectionView autoPinEdgesToSuperviewEdges]: unrecognized selector sent to instance，
      # 真机运行正常；
      #
      # 解决方法：
      # config.build_settings['EXCLUDED_ARCHS[sdk=iphonesimulator*]'] = 'arm64'
      # 或
      # Xcode设置为Rosetta 2模式
      
      config.build_settings['EXCLUDED_ARCHS[sdk=iphonesimulator*]'] = 'arm64'
    end
    
  end
end
