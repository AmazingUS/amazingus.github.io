一、三种枚举值、三个与界面旋转相关的方法

  • 1、UIDeviceOrientation

    • 当前设备的旋转方向（以home键的位置作为参照），不可以设置，一共有7种情况；

      case unknown
case portrait // Device oriented vertically, home button on the bottom
case portraitUpsideDown // Device oriented vertically, home button on the top
case landscapeLeft // Device oriented horizontally, home button on the right
case landscapeRight // Device oriented horizontally, home button on the left
case faceUp // Device oriented flat, face up
case faceDown // Device oriented flat, face down

    • 获取当前设备的旋转方向

      UIDevice.current.orientation

    • 监听当前设备的旋转方向

      // 最好先调用此方法（开始生成设备旋转通知），测试发现调用也会生成
UIDevice.current.beginGeneratingDeviceOrientationNotifications()

NotificationCenter.default.addObserver(self, selector: #selector(orientationDidChange), name: UIDevice.orientationDidChangeNotification, object: nil)

      NotificationCenter.default.removeObserver(self, name: UIDevice.orientationDidChangeNotification, object: self)
UIDevice.current.endGeneratingDeviceOrientationNotifications()

      @objc func orientationDidChange() {
    print("UIDevice Orientation: \(UIDevice.current.orientation.rawValue)")
}

  • 2、UIInterfaceOrientation

    • 当前程序界面的旋转方向，可以设置，一共有5种情况；

      case unknown
case portrait
case portraitUpsideDown
case landscapeLeft // 等价于设备的旋转方向landscapeRight
case landscapeRight // 等价于设备的旋转方向landscapeLeft
      • 需要注意一下，程序界面的左右旋转与设备的相反，因为设备向左旋转，程序界面需要先右旋转才能显示正；

  • 3、UIInterfaceOrientationMask

    • iOS 6开始增加的类型，是程序界面的旋转方向（UIInterfaceOrientation）的组合值；

      public static var portrait: UIInterfaceOrientationMask { get }
public static var landscapeLeft: UIInterfaceOrientationMask { get }
public static var landscapeRight: UIInterfaceOrientationMask { get }
public static var portraitUpsideDown: UIInterfaceOrientationMask { get }
public static var landscape: UIInterfaceOrientationMask { get }
public static var all: UIInterfaceOrientationMask { get }
// 不含有向下的方向
public static var allButUpsideDown: UIInterfaceOrientationMask { get }

  • 4、iOS 6开始新增三个与界面旋转相关的方法（计算属性）：

    open var shouldAutorotate: Bool { get }
    • 控制是否允许当前界面支持旋转；
    • 默认值为ture，允许旋转；
    open var supportedInterfaceOrientations: UIInterfaceOrientationMask { get }
    • 设置当前界面支持的所有方向，类型为UIInterfaceOrientationMask，便于表达支持多方向旋转的情况；
    • iPhone的默认值为allButUpsideDown，iPad的默认值为all；
    open var preferredInterfaceOrientationForPresentation: UIInterfaceOrientation { get }
    • 若界面以Present方式展示时，进入该界面的默认方向，类型为UIInterfaceOrientation；
    • 默认值为父界面的当前方向；

二、界面的可旋转范围、界面的默认方向、界面是否允许旋转

  • 1、界面的可旋转范围由以下4个配置项决定：

    • 1）Target的Device Orientation配置

      • a. Devices为Universal时，显示的勾选状态是iPhone的配置；
      • b. 对iPhone，默认没有选中UpsideDown；对于iPad，默认选中了UpsideDown；
      • c. 四个值全不选，等价于全选；

    • 2）Info.Plist配置

      • a. 修改Target的Device Orientation，本质就是修改Info.Plist，程序运行时以Info.plist中的配置为准；
      • b. Device Orientation、Info.Plist，修改任意一处配置，另外一处会自动更新同步；
      • c. 四个值全不配置，等价于全部配置；

    • 3）Appdelegate的代理方法配置

      // 该方法的详细说明见API注释
func application(_ application: UIApplication, supportedInterfaceOrientationsFor window: UIWindow?) -> UIInterfaceOrientationMask
      • a. 若未实现该方法，则返回Info.Plist的配置；
      • b. 若实现该方法，则以返回当前设置，Info.Plist配置会被忽略；
      • c. Info.Plist配置的意义：控制启动时刻的方向；

    • 4）Root控制器或Presented控制器的supportedInterfaceOrientations配置

      • a. 若当前界面控制器是Presented的，则可旋转范围为当前界面控制器的supportedInterfaceOrientations配置与Appdelegate的代理方法配置的交集；
      • b. 若当前界面控制器不是Presented的，则可旋转范围为Root控制器的supportedInterfaceOrientations配置与Appdelegate的代理方法配置的交集；

  • 2、界面的默认方向

    • 1）若当前界面控制器是Presented的，则界面的默认方向只由当前界面控制器的preferredInterfaceOrientationForPresentation配置决定，与界面的可旋转范围、界面是否允许旋转没有关系；
    • 2）若当前界面控制器不是Presented的，则界面的默认方向由当前界面的可旋转范围（优先级高）、设备的方向共同决定；

  • 3、界面是否允许旋转

    • 界面是否允许旋转，一般由以下2个配置项共同决定：
      • 1）是否打开设备自动旋转功能；
      • 2）Root控制器或Presented控制器的是否允许旋转shouldAutorotate配置；
    • 注意：代码强制旋转情况下的界面是否允许旋转，只需要判断是否允许旋转shouldAutorotate配置，不需要判断是否打开设备自动旋转功能；

三、两种触屏幕旋转的发方式

  • 1、代码强制旋转

    let orientation = UIInterfaceOrientation.landscapeRight
UIDevice.current.setValue(orientation.rawValue, forKey: "orientation")
UIViewController.attemptRotationToDeviceOrientation()
    • 能够发生旋转的必要条件：
      • a. 允许旋转shouldAutorotate返回true；
      • b. 旋转方向在可旋转范围内；
    • 常用情景：用户关闭了设备自动旋转功能，可以使用此方法触发屏幕旋转；

  • 2、手动旋转设备

    • 能够发生旋转的必要条件：
      • a. 打开设备自动旋转功能；
      • c. 允许旋转shouldAutorotate返回true；
      • b. 旋转方向在可旋转范围内；

四、场景分析

  • 1、让每个控制器都自己控制界面的旋转；

    • 由于界面是否允许旋转、界面的可旋转范围都与Root控制器或Presented控制器有关系，当界面结构为“UITabBarController -> UINavigationController -> UIViewController”时，可以使用下面的代码解决：

      // BaseTabBarController
override var shouldAutorotate: Bool {
    return self.selectedViewController?.shouldAutorotate ?? false
}

override var supportedInterfaceOrientations: UIInterfaceOrientationMask {
    return self.selectedViewController?.supportedInterfaceOrientations ?? .portrait
}

      // BaseNavigationController
override var shouldAutorotate: Bool {
    return self.topViewController?.shouldAutorotate ?? false
}

override var supportedInterfaceOrientations: UIInterfaceOrientationMask {
    return self.topViewController?.supportedInterfaceOrientations ?? .portrait
}

      // UIViewController
override var shouldAutorotate: Bool {
    return true
}

override var supportedInterfaceOrientations: UIInterfaceOrientationMask {
    return .all
}

  • 2、某些少量界面允许界面的旋转，其余的不允许旋转；

    • 方法1：
      • Appdelegate的代理方法配置放开旋转；
      • Root控制器限制旋转；
      • Presented控制器放开旋转；
      • 某些少量界面以Presented形式存在；
    • 方法2：
      • Appdelegate的代理方法配置放开旋转；
      • 使用上面的让每个控制器都自己控制界面的旋转方法；
      • 自定义BaseViewController限制旋转；
      • 某些少量界面重写相关方法放开旋转；
    • 方法3：
      • 直接在Appdelegate的代理方法配置中，判断当前界面是否是某些少量界面，进而限制、放开旋转；
