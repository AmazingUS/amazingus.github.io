本地使用多个SSH Key

2017/7/16
  • 一、概述
  • 二、使用多个SSH Key
  • 三、.ssh/config文件

一、概述

  • 执行 ssh-keygen -t rsa -C "xx@email.com"，回车确认，默认在~/.ssh/目录中生成名称为“id_rsa”的private key和“id_rsa_pub”的public key；注意：每次执行生成的key是不一样的；
  • 对于所有网站（如github、git.oschina等），一般使用同一个key进行认证；同一key在同一网站上只能用于一个账号，若需要使用不同的key就需要配置.ssh/config文件；
  • 使用不同的key可以认证不同网站的账号，也可以认证同一网站的不同账号；

二、使用多个SSH Key

  1. 生成第一个key；

    ## 指定生成的key的名称
ssh-keygen -t rsa -C "a@email.com" -f ~/.ssh/A_KEYNAME

  2. 生成第二个key；

    ssh-keygen -t rsa -C "b@email.com" -f ~/.ssh/B_KEYNAME

  3. 配置.ssh的config文件；

    vi ~/.ssh/config

    Host AA # 要匹配的别名
Hostname XX.com # 使用的域名
IdentityFile ~/.ssh/A_KEYNAME # 使用的Key

Host BB 
Hostname XX.com
IdentityFile ~/.ssh/B_KEYNAME

  4. 测试；

    ssh -T git@AA # 注意：git@别名
ssh -T git@BB

  5. 修改repo的ssh地址；

    • github repo的ssh地址结构：git@github.com:USERNAME/REPO_NAME.git

    • 比如git@github.com:twbs/bootstrap.git需要A_KEY管理，git@github.com:hexojs/hexo.git需要B_KEY管理；

      • 将ssh地址中的域名改为对应账号使用的域名别名，这样就可以使用该账号对应的key了；

        # 修改后的ssh地址
git@AA:twbs/bootstrap.git
git@BB:hexojs/hexo.git

三、.ssh/config文件

  • 补充：

    • a. 可以让config的Host和Hostname相同，这样不修改repo的ssh地址，就使用对应的key；比如让Host和Hostname都是“github.com”；
    • b. Host可以配置通配符*，规则写在最后，匹配前面没有匹配上的别名；
    • c. 使用ssh -T git@AA进行测试时，加-v参数会输出测试log，如ssh -T -v git@AA ；
    • d. .ssh/config文件可以加User参数；对Git服务，User为git，这样 git@别名 就可以简写成 别名 了，如 ssh -T git@AA 简写成 ssh -T AA ；

  • 示例

    Host blog
User git
Hostname github.com
IdentityFile ~/.ssh/me
# example: git clone blog:usera/repo.git

Host back
User git
Hostname gitee.com
IdentityFile ~/.ssh/me
# example: git clone back:usera/repo.git

# "Host" same as "URL"
Host gitee.com
Hostname gitee.com
IdentityFile ~/.ssh/me
# example: git clone git@gitee.com:usera/repo.git

# other "Host"
Host *
IdentityFile ~/.ssh/us
# example: git clone git@github.com:usera/repo.git、
#          ssh...
