一、概述、安装

  • Redis是一个key-value存储系统，与和Memcached类似；
    • 其支持存储的value类型有string(字符串)、list(链表)、set(集合)、zset(有序集合)等，并且对这些数据类型支持原子性的push/pop、add/remove、交并和差集等操作；
    • 为了保证效率，数据先缓存在内存中，之后会周期性地将更新的数据写入磁盘或把修改操作写入追加的记录文件中，进而可以实现了master-slave(主从)同步；

Nginx配置使用（Linux）

  • 一、概述、安装
  • 二、静态网页与动态网页
  • 三、配置文件nginx.conf
  • 四、虚拟主机匹配及禁止直接访问IP

一、概述、安装

  • Nginx是一个免费的开源高性能服务器，其有以下主要功能：静态HTTP服务器、 反向代理服务器、负载均衡、虚拟主机等，其中负载均衡、虚拟主机是基于反向代理实现的；（参考http://xxgblog.com/2015/05/17/nginx-start/）
    • a. Nginx可以将服务器上的静态文件（如HTML、图片）通过HTTP协议返回给客户端；
    • b. 客户端本来可以直接访问某网站的应用服务器，若在这中间加上Nginx，则客户端先请求Nginx，Nginx再请求应用服务器，此时Nginx就是反向代理服务器；
    • c. 当网站访问量非常大时，可以将相同的应用部署在多台服务器上，使用Nginx反向代理，让大量用户的请求分配给多台机器处理，实现负载均衡；
    • d. 网站访问量大需要负载均衡，网站访问量小，为节省成本，可以将多个网站部署在同一台服务器上（通过配置文件中的server_name来区分）；如将www.aaa.com、www.bbb.com两个网站部署在同一台服务器上，两个域名解析到同一个IP，但用户通过两个域名却打开两个完全不同的网站，互相不影响，就像访问两个服务器一样，所以叫两个虚拟主机；

一、内核管理

  • 清理多余的Linux内核

    • 当系统升级的时，旧的内核版本并不会被清理，之后就会导致/boot空间不足；卸载旧的内核版本可解决此问题；

      // 查看系统已经安装的内核版本
// 状态install说明已安装的版本
dpkg --get-selections | grep linux
dpkg --list | grep linux

// 查看系统当前使用的内核版本
uname -r

      // 使用purge命令删除相应的内核
// Ubuntu内核镜像包含: linux-headers-(内核头文件)、linux-image-(内核镜像)、linux-image-extra-（额外的内核模块）
apt-get purge linux-image-4.4.0-103-generic
// 或
apt-get purge linux-image-4.4.0*

// 若删除时，出现类似以下错误
// “linux-headers-generic : Depends: linux-headers
// -XXX-generic but it is not going to be installed”
// 进入/boot目录，将把4.4.0版本的config-4.4.0-*、vmlinuz-4.4.0-*、
// initrd.img-4.4.0-*、abi-4.4.0-*、System.map-4.4.0-*、retpoline-4.4.0-*都删除
// 然后执行apt-get install -f命令进行修复
// 最后，再次使用purge命令删除相应的内核

// 若GRUB配置在删除旧内核后没有正确升级，执行以下命令更新配置
update-grub2 或 update-grub

一、散列算法、加密算法、Base64

  • 一）散列算法

    • 又称为摘要算法，用于信息的签名、防篡改；运算结果不可逆；
    • 常见算法有MD5（将任意长度的内容散列成128bit）、SHA1（将任意长度的信息散列成160bit）、HMAC（Hash-based Message Authentication Code，需要输入一个密钥和一个信息，生成一个信息摘要）等；

  • 二）加密算法分类

    • a. 对称加密：

      • 加密和解密使用同一密钥，加密速度快，适用于大量内容的加密；
      • 常见算法有DES（Data Encryption Standard）、3DES（Triple DES）、AES（Advanced Encryption Standard）等；

    • b. 非对称加密：

      • 加密和解密使用不同的两个密钥，分别为公钥和私钥，并且需要配对使用，加密速度比较慢，适用于小量内容的加密；
      • 常见算法有RSA、DSA（Digital Signature Algorithm）、ECC（Elliptic Curves Cryptography）等；
      • 使用公钥加密私钥加密，RSA每次能够运算的数据长度不能超过RSA密钥长度，当超过时需要分段处理；
