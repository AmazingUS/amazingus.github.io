一、内核管理

  • 清理多余的Linux内核

    • 当系统升级的时，旧的内核版本并不会被清理，之后就会导致/boot空间不足；卸载旧的内核版本可解决此问题；

      // 查看系统已经安装的内核版本
// 状态install说明已安装的版本
dpkg --get-selections | grep linux
dpkg --list | grep linux

// 查看系统当前使用的内核版本
uname -r

      // 使用purge命令删除相应的内核
// Ubuntu内核镜像包含: linux-headers-(内核头文件)、linux-image-(内核镜像)、linux-image-extra-（额外的内核模块）
apt-get purge linux-image-4.4.0-103-generic
// 或
apt-get purge linux-image-4.4.0*

// 若删除时，出现类似以下错误
// “linux-headers-generic : Depends: linux-headers
// -XXX-generic but it is not going to be installed”
// 进入/boot目录，将把4.4.0版本的config-4.4.0-*、vmlinuz-4.4.0-*、
// initrd.img-4.4.0-*、abi-4.4.0-*、System.map-4.4.0-*、retpoline-4.4.0-*都删除
// 然后执行apt-get install -f命令进行修复
// 最后，再次使用purge命令删除相应的内核

// 若GRUB配置在删除旧内核后没有正确升级，执行以下命令更新配置
update-grub2 或 update-grub

一、散列算法、加密算法、Base64

  • 一）散列算法

    • 又称为摘要算法，用于信息的签名、防篡改；运算结果不可逆；
    • 常见算法有MD5（将任意长度的内容散列成128bit）、SHA1（将任意长度的信息散列成160bit）、HMAC（Hash-based Message Authentication Code，需要输入一个密钥和一个信息，生成一个信息摘要）等；

  • 二）加密算法分类

    • a. 对称加密：

      • 加密和解密使用同一密钥，加密速度快，适用于大量内容的加密；
      • 常见算法有DES（Data Encryption Standard）、3DES（Triple DES）、AES（Advanced Encryption Standard）等；

    • b. 非对称加密：

      • 加密和解密使用不同的两个密钥，分别为公钥和私钥，并且需要配对使用，加密速度比较慢，适用于小量内容的加密；
      • 常见算法有RSA、DSA（Digital Signature Algorithm）、ECC（Elliptic Curves Cryptography）等；
      • 使用公钥加密私钥加密，RSA每次能够运算的数据长度不能超过RSA密钥长度，当超过时需要分段处理；

一、is、==、对象的布尔值（if ObjA）、None、not

  • 1. is：

    • 比较两个对象的内存地址(id(a))是否相同，若是相同则返回True，否则为False；

    • 注意：由于Python会对较小的对象进行缓存（比如str、int等，自定义类不会进行缓存），所以有时并不会创建新的对象，而是从缓存中进行查找返回，没有才创建；

    • is not表示is的相反结果，内存地址不相同时返回True；

Java、ObjC、Swift中的错误、异常的处理总结

  • 一、Java
  • 二、ObjC
  • 三、Swift

一、Java

  • 在Java中，所有的错误、异常的最上层父类为Throwable，其有两个重要的子类：Error（错误）、Exception（异常）；

    • a. Error（错误）：程序无法处理，大多数为JVM出现的问题；比如JVM运行错误（VirtualMachineError）、当 JVM没有内存资源错误（OutOfMemoryError）等；
    • b. Exception（异常）：程序可以处理，这是与Error（错误）的本质区别；

  • Exception（异常）也分为两种：运行时异常、非运行时异常（编译异常）；

    • a. 运行时异常：

      • a). 在编译时不会被检查；一般是由程序逻辑问题引起的，应该从逻辑角度尽可能避免这类异常的发生；
      • b) 程序中可以选择处理（try-catch-finally、throws），或者不处理异常；
      • c) 其都是RuntimeException及其子类，比如NullPointerException（空指针异常）、IndexOutOfBoundsException（下标越界异常）等；

    • b. 非运行时异常（编译异常）：

      • a) 必须要进行处理（try-catch-finally、throws）的异常，若不处理，程序就不能编译通过；比如IOException、SQLException等
      • b）除RuntimeException及其子类以外的异常都是非运行时异常（编译异常）；

  • 通常也将可以把Java的错误、异常分为：可查的异常（checked exceptions，也就是上面的非运行时异常）、不可查的异常（unchecked exceptions，包含错误和运行时异常）；

  • 处理异常有两种方式：捕获异常（try-catch-finally）、捕获异常后再抛出（throws）；
