001、is、==、对象的布尔值（if ObjA）、None、not

  • 1、is：

    • 比较两个对象的内存地址(id(a))是否相同，若是相同则返回True，否则为False；

    • 注意：由于Python会对较小的对象进行缓存（比如str、int等，自定义类不会进行缓存），所以有时并不会创建新的对象，而是从缓存中进行查找返回，没有才创建；

    • is not表示is的相反结果，内存地址不相同时返回True；

      # case 1:
s1 = 'Hello'
s2 = 'Hello'  # 会使用缓存
print(id(s1))
print(id(s2))
print(s1 is s2)  # True

# case 2:
a = 2
b = 1 * 2  # 会使用缓存
print(id(a))
print(id(b)) 
print(a is b)  # True

# case 3:
print(1000 is 10 ** 3)  # False，不会使用缓存

# case 4:
class A:
    pass

print(A() is A()) # False

  • 2、==：

    • 返回对象的__eq__函数的结果（布尔值）；

    • ==、!=、>、<等值比较都可以通过重写__eq__、__ne__等函数来改变；

    • str的__eq__函数会比较字符串的内容是否一致；自定义类的__eq__函数会返回父类的默认值；

    • **若需要重写自定义类的__eq__，一般也需要重写其__hash__，否则该自定义类不可以在哈希集合中使用；**若没有重写__eq__，则此时不应该重写__hash__；对象放到哈希集合中时，先调用__hash_进行定位其集合中的位置，再调用__eq__决定是否应该将该对象访问集合中；

  • 3、对象的布尔值（if ObjA）：

    • 当一个对象的__bool__()函数返回True并且__len()__不返回0时，该对象的布尔值为True，否则为False； 比如0、0.0、0L、""（空字符串）、[]、()、{}、False、None等；

    • 注意：None的布尔值为False；

    • 对象的布尔值在if判断、布尔运算时（and、or、not）会用到；

  • 4、None：

    • 一个特殊的常量，含义为空，相当于C语言中的NULL，其可以赋值给任意变量；None的数据类型为NoneType；

    • 注意：空列表、空字符串、空字典都不是None；

  • 5、not：

    • 布尔运算符非，可以和is、in连用组成is not、not in语句；

    • Python的布尔运算符使用的是not而不是!，关系运算符!=（不等于）才用到了!；

  • 6、举例：

    • if a == None：（不常用，写法不Pythonic）判断a是否为空，等价于if a is None；
    • if a != None：（不常用，写法不Pythonic）判断a是否不为空，等价于if a is not None；
    • if a is None：判断a是否为空；None只有一个，不存在另一个值为None但id(None)不相等的对象；
    • if a is not None：判断a是否不为空；
    • if a：判断对象的布尔值是否为True；
    • if not a：判断对象的布尔值是否为False；
    • 注意：当判断一个对象是否为None时，使用if a is None和if not a是没有区别；当判断一个对象是否False时，就只使用if not a；

003、if __name__ == 'main'作用

  • 一个Python文件一般有两种使用方式，第一种是作为脚本直接执行，第二种是import到其他的Python脚本中，被调用执行；if __name__ == 'main'就是用来判断是否是以第一种方式运行（脚本直接运行）；

  • 每个Python模块（Python文件）都包含内置的变量__name__，当以脚本直接执行时，该变量值为字符串“main”，当以被import被引入调用执行时，该变量值为不含有后缀名的Python文件名；

    # test_a.py
print('1', __name__)

if __name__ == '__main__':
    print('2', __name__)

# 测试：作为脚本直接执行

# 输出：
1 __main__
2 __main__

    # test_a_import.py
import test_a

print('3', __name__)

if __name__ == '__main__':
    print('4', __name__)

# 测试：import到其他的python脚本中，被调用执行

# 输出：
1 test_a
3 __main__
4 __main__

004、from __future__ import *作用

  • 1、概述：

    • from __future__ import *，该代码总是放到代码最前面；
    • _future__模块功能：将Python 3的特性移植到Python 2中；引入后，不兼容的老版本Python 2的特性将会被禁用；

  • 2、使用：

    # 1、整数除法

# Python 2.x
5/2
>>> 2

# Python 2.x（from __future__ import division）
5/2
>>> 2.5

    # 2、绝对引入

# 主要是针对Python 2.4及之前的版本的；
# 这些版本在引入某一个.py文件时，会首先从当前目录下，查找是否有该文件；
# 若有，则优先引用当前目录下的文件；而若想引用Python自带的同名的.py文件时，则需要使用from __future__ import absolute_import；

    # 3、print函数
# Python 2中引入该模块之后便具有Python 3的功能，此时Python2原有的的print表示形式将被替代；

# Python 2.x
print "Hello"

# Python 2.x（from __future__ import print_function）
print("Hello")
print "World"

>>> print "World"
  File "<stdin>", line 1
    print "World"
                      ^
SyntaxError: invalid syntax

    # 4、with用法

# Python 2.x
try:
    with open('test.txt', 'w') as f:
    f.write('Hello World')
finally:
    f.close()

# Python 2.x（from __future__ import with_statement）
# 不用再写 异常检测代码
with open('test.txt', 'w') as f:
    f.write('Hi there!')

005、NumPy的np.reshape()、obj.reshape()、np.resize()、obj.resize()方法区别

  • 1、np.reshape

    # np.reshape
# 不改变原数组，返回值返回结果
# 只能改变形状，无法扩展size(2, 3, 10)、缩小size(2, 3, 1)

a = np.arange(24)
a1 = np.reshape(a, (2, 3, 4))
print('a:\n', a)
print('a1:\n', a1)

  • 2、obj.reshape

    # obj.reshape
# 不改变原数组，返回值返回结果
# 只能改变形状，无法扩展size(2, 3, 10)、缩小size(2, 3, 1)

b = np.arange(24)
b1 = b.reshape((2, 3, 4))
print('b:\n', b)
print('b1:\n', b1)

  • 3、np.resize

    # np.resize
# 不改变原数组，返回值返回结果
# 可以改变形状，可以扩展size(2, 3, 10)、缩小size(2, 3, 1)
# 扩展默认值为原数据从头开始顺序填充

c = np.arange(24)
c1 = np.resize(c, (2, 3, 4))
print('c:\n', c)
print('c1:\n', c1)

  • 4、obj.resize

    # obj.resize
# 改变原数组，返回值返回None
# 可以改变形状，可以扩展size(2, 3, 10)、缩小size(2, 3, 1)
# 扩展默认值为0

d = np.arange(24)
d1 = d.resize((2, 3, 4))
print('d:\n', d)
print('d1:\n', d1)

006、Python中的反转[::-1]、[:,:,::-1]

  • 1、每一维度的中间有两个冒号，即[a:b:c]，意思为从a到b，间隔为c；

    • 若省略c，可以写成a:b；
    • 若省略a、b、c，可以写成::或:；
    • 若省略多个::，可以写成...；

  • 2、一维数组的反转

    import numpy as np

a = np.arange(1, 10)
print('a:\n', a)
print('a[8:1:-1]:\n', a[8:1:-1])

# 输出
a:
 [1 2 3 4 5 6 7 8 9]
a[8:1:-1]:
 [9 8 7 6 5 4 3]

  • 3、多维数组的反转：需要分别确定反转的维度数

    import numpy as np

a = np.arange(24)
a = np.reshape(a, (2, 3, 4))

print('a:\n', a)

# 第1、2维不变，第3维反转
print('a[::, ::, ::-1]:\n', a[::, ::, ::-1])

# 第1、3维不变，第2维反转
print('a[::, ::-1, ::]:\n', a[::, ::-1, ::])

# 第2、3维不变，第1维反转
print('a[::-1, ::, ::]:\n', a[::-1, ::, ::])

# 第1、2维不变，第3维反转（简写）
print('a[:, :, ::-1]:\n', a[:, :, ::-1])

# 第1、2维不变，第3维反转（简写）
print('a[..., ::-1]:\n', a[..., ::-1])
    • a[::, ::, ::-1]、a[:, :, ::-1]、a[..., ::-1]：常用于图片通道转换（BGR转RGB，或RGB转BGR）；

007、MatplotlibDeprecationWarning警告

  • 1、概述：

    • 1）使用Matplotlib的show方法时，出现警告：MatplotlibDeprecationWarning: Support for FigureCanvases without a required_interactive_framework attribute was deprecated in Matplotlib 3.6 and will be removed two minor releases later.

    • 2）原因：

      • Matplotlib 3.6开始，对于没有required_interactive_framework属性的Figure Canvases（绘图画布），出现警告（之后会移除支持）；

    • 3）解决办法1：

      • 设置 => Pycharm科学模式（Python Scientific）=> 取消勾选Show plots in tool window；
      • Show plots in tool window：表示Matplotlib的绘图界面，会在Pycharm中以工具栏窗口显示（IDE右侧），否则弹出一个“独立的”绘图窗口；

    • 4）解决办法2：

      • 更新plt.show()代码，修改为更现代的方法；
      • 在交互式环境（如Jupyter Notebook）中，使用%matplotlib命令，来配置交互式绘图；
      • 在脚本中，使用plt.pause()，或者将绘图保存到文件，而不是显示在屏幕上；
      • 若确实需要使用plt.show()来显示绘图，并且确定代码没有其他问题，可以忽略这个警告；

  • 2、参考：

    • Matplotlib show 方法警告原因：https://blog.csdn.net/wo541075754/article/details/130719942
    • Pycharm 中画图警告：MatplotlibDeprecationWarning：https://blog.csdn.net/qq_43585922/article/details/131491621

# 关闭X、Y轴坐标显示
plt.xticks([]), plt.yticks([]) # 方式1
# plt.axis('off') # 方式2

# 显示图片
plt.show()

011、PIL（pillow）不支持BGR、处理方式

  • 1、PIL（pillow）不支持BGR格式，支持1（二值图）、L（灰度图）、P、RGB、RGBA、CMYK、YCbCr、HSV等格式；并且创建时，默认使用RGB模式；

    • 相关文档：https://pillow.readthedocs.io/en/stable/handbook/concepts.html#concept-modes
    from PIL import Image
import numpy as np

img_w, img_h = 200, 200
np_arr = np.zeros((img_h, img_w, 3), dtype=np.uint8)
np_arr[100, 100] = [255, 0, 0]

img = Image.fromarray(np_arr) # 不传入mode参数，默认RGB；
# img = Image.fromarray(np_arr, mode='HSV')

print('mode:', img.mode)  # mode: RGB

img.show()

    import numpy as np
from PIL import Image
import cv2

img = Image.open('lena.jpg')
print(img.mode)  # RGB

# ❌，PIL（Pillow）不支持BGR格式
# img1 = image.convert("BGR")

# 1、转成BGR格式
arr = np.array(img)
arr = arr[..., ::-1]

# 2、PIL（Pillow）显示（图片颜色错误）
img1 = Image.fromarray(np.uint8(arr))
img1.show()
print(img1.mode)  # RGB，其实是BGR格式

# 3、转成RGB格式
arr2 = cv2.cvtColor(arr, cv2.COLOR_BGR2RGB)

# 4、PIL（Pillow）显示（图片颜色正确）
img2 = Image.fromarray(np.uint8(arr2))
img2.show()
print(img1.mode)  # RGB

012、requests的post()方法data、json参数区别，以及同时发送参数和上传多个文件

  • 1、POST请求编码方式

    • 1）application/x-www-form-urlencoded

      • 浏览器原生form表单；
      POST http://httpbin.org HTTP/1.1
Content-Type: application/x-www-form-urlencoded

a_test=112233&b_test=223344

    • 2）multipart/form-data; boundary=xx

      • 浏览器上传文件时所用的编码方式，通过boundary来分割不同字段的数据；
      POST http://httpbin.org HTTP/1.1
Content-Type:multipart/form-data; boundary=----WebKitFormBoundaryrGKCBY7qhFd3TrwA

------WebKitFormBoundaryrGKCBY7qhFd3TrwA
Content-Disposition: form-data; name="text"

text
------WebKitFormBoundaryrGKCBY7qhFd3TrwA
Content-Disposition: form-data; name="file"; filename="111.jpg"
Content-Type: image/png

PNG ... content of 111.jpg ... ... ... ... ...
------WebKitFormBoundaryrGKCBY7qhFd3TrwA--

    • 3）application/json

      • 由于JSON规范的流行，越来越多使用这种编码方式；该方式可以提交比较复杂的结构化数据，方便各种类型的数据交互；
      POST http://httpbin.org HTTP/1.1 
Content-Type: application/json

{"test":"test123","test22":[111,222,333]}

    • 4）text/xml

      • 一般用于使用XML作为编码方式的远程调用规范；
      POST http://httpbin.org HTTP/1.1 
Content-Type: text/xml

<?xml version="1.0"?>
<methodCall>
    <methodName>examples.getStateName</methodName>
    <params>
        <param>
            <value><i4>41</i4></value>
        </param>
    </params>
</methodCall>

  • 2、requests的post()方法data、json参数区别

    • 使用data参数时，发送的数据出现在了form表单字段中，Content-Type字段的值为application/x-www-form-urlencoded；
    • 使用json参数时，Content-Type字段的值为application/json；
    • 使用files参数（或者data和files、json和files）时，Content-Type字段的值为multipart/form-data; boundary=xx；
    • 同时使用json和files参数时，json中的参数被忽略；所以为了实现上传文件的同时，还传递其他参数应该使用data和files参数；

  • 3、同时发送参数和上传多个文件

        files.append(('img[]', (file_name, data, 'image/jpeg')))

014、Python命令行参数解析argparse

  • 1、使用步骤

    • 1）导入ArgumentParser；

      from argparse import ArgumentParser

    • 2）创建一个解析对象ArgumentParser；

      argumentParser = ArgumentParser(description=None)

    • 3）然后向该解析对象中，添加命令行参数和选项，每一个add_argument方法对应一个参数或选项；

      # 定义单个的命令行参数应当如何解析
argumentParser.add_argument(name or flags...[, action][, nargs][, const]
    [, default][, type][, choices][, required][, help][, metavar][, dest])
      • a. name or flags：一个命名或者一个选项字符串的列表，例如foo或-f、--foo。
      • b. action：当参数在命令行中出现时，使用的动作基本类型；
        • action='store'：存储参数的值，默认值的动作；
        • action='store_true'：表示传递该参数时，将其值赋值为True，否则False；
        • action='store_false'：表示传递该参数时，将其值赋值为False，否则True；
        • action='store_const'：表示传递该参数时，使用const值（必须设置），否则为None；
        • ...
      • c. nargs：设置参数可以提供的个数；
        • nargs=N（一个整数）：N个参数会被聚集到一个列表中；
        • nargs='*'：表示参数可设置零个或多个；
        • nargs='+'：表示参数可设置一个或多个；
        • nargs='?'：表示参数可设置零个或一个；
      • d. const：被一些action和nargs选择所需求的常数；
      • e. default：没有设置值情况下的默认参数；
      • f. type：参数类型；
      • g. choices：参数值只能从固定选项里选择；
      • h. required：表示这个参数是否一定需要设置（仅可选参数时有效）；
      • i. help：指定参数的说明信息；
      • j. metavar：在使用方法消息中使用的参数值示例；
      • k. dest：设置参数在代码中的变量名，默认是变量名是“--”或“-”后面的字符串；

    • 4）调用parse_args()方法进行解析，解析成功之后即可使用；

      args = parser.parse_args()
print(args)

  • 2、args分为可选参数（用“--”、“-”指定）、必选参数（不加“--”、“-”指定）；当“--”、“-”同时存在时，使用“--”的为参数名，但在命令行输入时，用其中任何一个指定都可以；

    # 可选参数
parser.add_argument('--work-dir', '-wd', help='the dir to save logs and models')

# 命令行执行，进行测试
python xx.py --work-dir WORK_DIR
# 或
python xx.py -wd WORK_DIR

    # 必选参数
parser.add_argument('config', help='train config file path')

# 命令行执行，进行测试
python xx.py config

  • 3、action参数案例

    import argparse

parser = argparse.ArgumentParser(description='Process "action" Test.')
parser.add_argument('-a', action='store_true')
parser.add_argument('-b', action='store_false')
parser.add_argument('-c', action='store_const', const=100)

args = parser.parse_args()
print(args)

# 命令行执行，进行测试
python xx.py
Namespace(a=False, b=True, c=None)

python xx.py -a
Namespace(a=True, b=True, c=None)

python xx.py -b 
Namespace(a=False, b=False, c=None)

python xx.py -c
Namespace(a=False, b=True, c=100)

  • 4、子命令案例

    import argparse

parser = argparse.ArgumentParser(description='Process "git" Test.')
subparsers = parser.add_subparsers(dest='command', help='sub command help')

# 添加clone子命令的解析器
parser_clone = subparsers.add_parser('clone', help='clone help')
parser_clone.add_argument('--repository', help='Repository to clone')

# 创建pull子命令的解析器
parser_pull = subparsers.add_parser('pull', help='pull help')
parser_pull.add_argument('--repository', help='Repository to pull')

args = parser.parse_args()

# 因为设置dest='command'，所以使用args.command
if args.command == 'clone':
    print(f"Cloning repository {args.repository}...")
elif args.command == 'pull':
    print(f"Pulling repository {args.repository}...")

# 命令行执行，进行测试
python xx.py clone --repository REPOSITORY
python xx.py pull --repository REPOSITOR
