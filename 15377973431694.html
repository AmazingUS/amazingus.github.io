Logrotate使用教程

2018/9/24
  • 一、概述
  • 二、Crontab（定时任务）
  • 三、配置案例
  • 三、其他日志切割方法

一、概述

  • Logrotate是一个非常有用的管理日志工具，其可以自动对日志进行切割（轮循）、压缩以及删除旧的日志文件；比如让设置logrotate自动化运作，让/var/log/acc日志文件每30天轮循，并删除超过6个月的日志；

  • Linux默认安装了Logrotate，其默认的配置文件为：/etc/logrotate.conf、/etc/logrotate.d/目录；

    • logrotate.conf为主要的配置文件，logrotate.d是一个目录，该目录中的所有文件默认都会被导入到/etc/logrotate.conf中（include /etc/logrotate.d），在logrotate.conf被logrotate时，同时也被logrotate；
    • 若/etc/logrotate.d/里面的文件中没有配置一些设置，则使用/etc/logrotate.conf中设置的默认值；
    • 对于多数Service，即使系统没有安装Logrotate，一个默认的logrotate配置文件会在安装程序包时自动创建；若是使用源代码编译手动安装，就需要手动在/etc/logrotate.d/目录中手动创建一个logrotate配置文件；
  • 日志切割原理（以/var/log/acc文件为例）
    • a. 第1次执行完rotate（轮循）之后，原本的acc会变成acc.1，并且生成一个空的acc存储新的日志；
    • b. 第2次执行之后，acc.1会变成acc.2，而acc会变成acc.1，又生成一个空的acc存储新的日志；
    • c. 若设定保留3个日志，那么执行第3次时，则acc.3这个文件就会被删除，并由之后较新的日志所取代；

二、Crontab（定时任务）

  • Logrotate是基于crontab来运行的，日志轮循是系统自动完成的；

    • 默认的logrotate被加入cron的/etc/cron.daily中作为每日任务执行，其脚本为/etc/cron.daily/logrotate（内容为logrotate logrotate.conf...；默认etc/logrotate.d/目录下的内容也被logrotate）
    • CRON表达式是一个表示时间周期的字符串；

  • 查看crontab定时任务

    cat /etc/crontab

  • 查看logrotate上次轮换日志文件的时间

    cat /var/lib/logrotate/status

  • 修改系统时间后，必须手动重启重启crontab、rsyslog服务（只是重启服务器是不行的），否则crontab日志时间与系统时间不一致；

    systemctl restart cron
systemctl restart rsyslog

三、配置案例

  • 需求：

    • 对放在/var/log/nginx下的nginx日志进行管理：access.log只记录当天的日志；之前的日志以access.log_时间命名，一共保留10天的日志；

  • 配置

    • 1、vim /etc/logrotate.d/nginx文件（注意：使用安装包安装nginx时，会自动创建该文件），内容如下：

      /var/log/nginx/*.log {
    daily
    rotate 100
    missingok
    notifempty
    dateext
    create 644 root root
    if [ -f /usr/local/nginx/logs/nginx.pid ]; then # 若nginx没启动则不执行该操作
        kill -USR1 `cat /usr/local/nginx/logs/nginx.pid`
        # 传递给它信号重新生成日志（不是中止Nginx的进程）
    fi
}

      • 参数说明

        daily：日志文件将按天轮循，值可以为daily、weekly、monthly、yearly
rotate 5：最多存储5个归档日志，对于第6个归档，删除时间最久的归档
compress：在轮循任务完成后，对已轮循的归档将使用gzip进行压缩，不需要则注释掉
delaycompress：必须配合compress使用，指示logrotate不要将最近的归档压缩，压缩将在下一次轮循周期进行
missingok：在日志轮循期间，错误将被忽略，如“文件无法找到”等错误
notifempty：若日志文件为空，轮循不会进行
create 644 root root：以指定的权限创建全新的日志文件，同时logrotate也会重命名原始日志文件
postrotate/endscript：在所有其它指令完成后，postrotate与endscript包含的命令将被执行；rsyslogd进程将立即再次读取其配置并继续运行

dateext：就是切割后的日志文件以当前日期为格式结尾，如xxx.log-20180924；默认切割出来是按数字递增，即 xxx.log-1这种格式
dateformat %s：必须配合dateext使用，定义文件切割后的文件名，支持%Y、%m 、%d 、%s四个参数

      • 验证logrotate脚本是否正确

        logrotate -d /etc/logrotate.d/nginx

      • 默认logrotate是通过crontab定期执行的，也可以手动执行查看结果

        logrotate -vf /etc/logrotate.d/nginx

    • 2、在crontab中添加定时脚本，让logrotate每天进行一次轮循；

      #crontab -e
0 0 * * * /usr/sbin/logrotate -f /etc/logrotate.d/nginx

三、其他日志切割方法

  • 除了利用Logrotate工具实现日志切割之外，还可以使用Python脚本、Shell脚本以实现日志切割

  • 对nginx日志进行切割

    • 1、创建Python脚本（vim /nginx_log_rotate.py）

      import datetime, os, shutil

log_path = '/var/log/nginx/'
log_file = 'access.log'
yesterday = (datetime.datetime.now() - datetime.timedelta(days=1))

try:
    os.makedirs(log_path + yesterday.strftime('%Y') + os.sep + yesterday.strftime('%m'))
except Exception as e:
    print(e)

shutil.move(log_path + log_file, log_path
            + yesterday.strftime('%Y') + os.sep
            + yesterday.strftime('%m') + os.sep
            + log_file + '_' + yesterday.strftime('%Y%m%d') + '.log')

os.popen("sudo kill -USR1 `cat /usr/local/nginx/logs/nginx.pid`")

    • 2、添加定时任务

      30 0 * * * /usr/bin/python /nginx_log_rotate.py > /dev/null 2>&1
